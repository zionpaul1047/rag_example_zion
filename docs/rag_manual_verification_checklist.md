# RAG 수동 검증 체크리스트

이 문서는 채팅, 이전 대화, 세션 파일, 관리 문서, 삭제, RAG 진단 표시가 실제 앱에서 기대대로 동작하는지 확인하기 위한 체크리스트입니다.

## 실행 전 준비

- Backend 서버가 실행 중인지 확인한다.
- Frontend 개발 서버가 실행 중인지 확인한다.
- 일반 사용자 계정과 관리자 계정으로 각각 로그인할 수 있는지 확인한다.
- `.env`의 `LOG_LEVEL`이 `INFO` 이상으로 설정되어 있는지 확인한다.

## 1. 새 채팅

- 채팅 메뉴에서 `새 대화`를 누른다.
- 일반 질문을 전송한다.
- 답변이 생성되고 `conversation_id`가 표시되는지 확인한다.
- AI 답변 아래 `응답 정보`를 펼친다.
- 요청 모델, 실제 사용 모델, 검색 평가, 재검색 횟수, graph trace가 표시되는지 확인한다.
- 오른쪽 `참조` 패널에서 참조 문서와 검색 진단이 표시되는지 확인한다.

기대 결과:
- 새 대화가 생성된다.
- 답변이 메시지 목록에 남는다.
- `응답 정보`, `참조`, `검색 진단`이 같은 답변 기준으로 표시된다.

## 2. 메뉴 이동 후 채팅 복귀

- 채팅에서 질문과 답변을 만든다.
- 다른 메뉴로 이동한다.
- 다시 채팅 메뉴로 돌아온다.

기대 결과:
- 현재 대화의 메시지가 유지되거나, 선택된 대화 기준으로 다시 로드된다.
- 마지막 AI 답변의 참조와 검색 진단이 오른쪽 패널에 다시 표시된다.

## 3. 내 대화에서 이전 대화 이어하기

- `내 대화` 메뉴로 이동한다.
- 방금 생성한 대화가 목록에 표시되는지 확인한다.
- 해당 대화를 선택한다.
- 채팅 메뉴로 이동하거나 이어서 대화하기 동작을 수행한다.
- 이전 메시지 전체가 채팅 화면에 표시되는지 확인한다.
- 추가 질문을 전송한다.

기대 결과:
- 이전 대화가 현재 채팅 화면에 복원된다.
- 새 질문은 같은 `conversation_id`로 이어진다.
- 기존 참조 metadata와 새 답변 metadata가 모두 표시된다.

## 4. 세션 파일 기반 RAG

- 일반 사용자로 로그인한다.
- 세션 파일 관리에서 테스트 문서를 업로드한다.
- 채팅에서 업로드한 문서 내용과 직접 관련된 질문을 한다.
- 참조 패널에서 `session` 유형 source가 표시되는지 확인한다.
- `content_preview`가 실제 문서 조각과 맞는지 확인한다.

기대 결과:
- 세션 문서가 검색 후보에 포함된다.
- 검색 진단의 `세션 문서` 값이 1 이상이다.
- 다른 사용자로 로그인하면 해당 세션 문서가 검색되지 않는다.

## 5. 관리 문서 기반 RAG

- 관리자 계정으로 로그인한다.
- RAG 문서 관리에서 문서를 등록하고 인덱싱한다.
- 채팅에서 관리 문서 내용과 관련된 질문을 한다.
- 참조 패널에서 managed source가 표시되는지 확인한다.

기대 결과:
- 관리 문서가 KB 검색 후보에 포함된다.
- 검색 진단의 `KB 검색` 값이 1 이상이다.
- 참조 source의 preview가 등록한 문서 내용과 맞다.

## 6. 문서 삭제 후 참조 제외

- 세션 파일 문서를 삭제한다.
- 삭제한 세션 문서 내용으로 다시 질문한다.
- 관리 문서를 완전 삭제한다.
- 완전 삭제한 관리 문서 내용으로 다시 질문한다.

기대 결과:
- 삭제된 세션 문서는 더 이상 참조되지 않는다.
- 완전 삭제된 관리 문서는 vector/keyword 검색 결과에서 제외된다.
- 활성 managed 문서를 삭제한 경우 이전 retired 버전 복원 정책이 기대대로 동작한다.

## 7. no_docs / retry 흐름

- 문서와 관련 없는 질문을 한다.
- 낮은 관련도 질문으로 retry가 발생할 수 있는 케이스를 시도한다.
- AI 답변의 `응답 정보`를 펼친다.

기대 결과:
- 참조가 0개여도 `진단` 버튼 또는 검색 진단이 확인된다.
- retry가 발생한 경우 `재검색 쿼리`가 표시된다.
- graph trace에 `retry_retrieve` 또는 `no_docs_answer`가 표시된다.

## 8. Backend 로그 확인

- Backend 콘솔 또는 로그 수집 위치에서 `rag_summary` 로그를 확인한다.

기대 결과:
- 로그에 `conversation_id`, `source_count`, `session_document_count`, `kb_result_count`, `eval_result`, `requested_provider`, `used_provider`, `retry_count`, `graph_trace`가 포함된다.
- stream/non-stream 응답 모두 완료 시점에 로그가 남는다.

## 최종 통과 기준

- 주요 채팅 흐름에서 대화가 사라지지 않는다.
- 내 대화에서 이전 대화를 불러와 이어서 대화할 수 있다.
- 세션 문서는 사용자별로 격리된다.
- 삭제된 문서는 다시 참조되지 않는다.
- RAG 참조, 검색 진단, 응답 정보, backend 로그가 같은 흐름을 설명한다.
